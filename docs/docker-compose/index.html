Docker Compose Configuration

Complete examples for running Bun + Infisical with Docker Compose. From basic setups to multi-service configurations with databases.

Basic docker-compose.yml

A minimal configuration to get started with the Bun + Infisical image. This example runs a simple Bun application without external dependencies. 

version: '3.8'

services:
  app:
    image: sswork/bun-infisical:latest
    container_name: bun-app
    working_dir: /app
    volumes:
      - ./:/app
    ports:
      - "3000:3000"
    command: bun run start

With Infisical Token Authentication

Configure your application to securely fetch secrets from Infisical at runtime. The INFISICAL_TOKEN is required for authentication. 

version: '3.8'

services:
  app:
    image: sswork/bun-infisical:latest
    container_name: bun-app-secure
    working_dir: /app
    volumes:
      - ./:/app
    ports:
      - "3000:3000"
    environment:
      - INFISICAL_TOKEN=${INFISICAL_TOKEN}
      - INFISICAL_API_URL=${INFISICAL_API_URL:-https://app.infisical.com}
    command: infisical run -- bun start

Required Environment Variables

  • INFISICAL_TOKEN - Your Infisical service token for authentication
  • INFISICAL_API_URL - Infisical API endpoint (optional, defaults to https://app.infisical.com)

With Environment Variables

Load environment variables from a .env file and combine them with Infisical secrets. This approach keeps sensitive configuration out of your compose file. 

version: '3.8'

services:
  app:
    image: sswork/bun-infisical:latest
    container_name: bun-app-env
    working_dir: /app
    volumes:
      - ./:/app
    ports:
      - "3000:3000"
    env_file:
      - .env
    environment:
      - NODE_ENV=production
      - INFISICAL_TOKEN=${INFISICAL_TOKEN}
    command: infisical run -- bun start

Example .env File

# Application
NODE_ENV=production
PORT=3000
LOG_LEVEL=info

# Infisical
INFISICAL_TOKEN=your-service-token-here
INFISICAL_API_URL=https://app.infisical.com

# Database (if using external)
DATABASE_URL=postgresql://user:pass@db:5432/myapp

Multi-Service Setup (App + Database)

A complete production-ready setup with a PostgreSQL database, Bun application, and persistent volumes. Includes health checks and proper service dependencies. 

version: '3.8'

services:
  db:
    image: postgres:16-alpine
    container_name: app-database
    restart: unless-stopped
    environment:
      POSTGRES_USER: ${DB_USER:-appuser}
      POSTGRES_PASSWORD: ${DB_PASSWORD:-secretpassword}
      POSTGRES_DB: ${DB_NAME:-myapp}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USER:-appuser} -d ${DB_NAME:-myapp}"]
      interval: 10s
      timeout: 5s
      retries: 5

  app:
    image: sswork/bun-infisical:latest
    container_name: bun-app-full
    restart: unless-stopped
    working_dir: /app
    depends_on:
      db:
        condition: service_healthy
    volumes:
      - ./:/app
      - node_modules:/app/node_modules
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DATABASE_URL=postgresql://${DB_USER:-appuser}:${DB_PASSWORD:-secretpassword}@db:5432/${DB_NAME:-myapp}
      - INFISICAL_TOKEN=${INFISICAL_TOKEN}
      - INFISICAL_API_URL=${INFISICAL_API_URL:-https://app.infisical.com}
    env_file:
      - .env
    command: infisical run -- bun start
    healthcheck:
      test: ["CMD", "wget", "-q", "--spider", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

volumes:
  postgres_data:
    driver: local
  node_modules:
    driver: local

Running Multi-Service Setup

# Start all services
docker-compose up -d

# View logs
docker-compose logs -f app

# Stop all services
docker-compose down

# Stop and remove volumes
docker-compose down -v

Common Docker Compose Commands

Command Description
docker-compose up -d Start services in detached mode
docker-compose down Stop and remove containers
docker-compose logs -f Follow log output
docker-compose ps List running containers
docker-compose exec app sh Open shell in app container
docker-compose restart app Restart specific service
