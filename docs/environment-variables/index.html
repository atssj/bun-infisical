Environment Variables

Complete reference for configuring the Bun + Infisical Docker image. Learn how to authenticate with Infisical and customize your deployment.

Overview

The Bun + Infisical Docker image uses environment variables for configuration and authentication. These variables control how the Infisical CLI connects to your secrets backend and authenticates your application.

Required Setup

At minimum, you need an INFISICAL_TOKEN to authenticate with the Infisical API. This token grants your container access to the secrets stored in your Infisical projects.

Environment Variable Reference

Variable Required Default Description
INFISICAL_TOKEN Required None Your Infisical service token for authentication. This token is used to fetch secrets from your Infisical projects. Learn more about service tokens.
INFISICAL_API_URL Optional https://app.infisical.com Custom Infisical API URL for self-hosted instances. Use this when running your own Infisical server instead of the cloud service.

Usage Examples

Docker Run

Run a container with environment variables passed directly:

docker run -e INFISICAL_TOKEN=your-service-token \
  -e INFISICAL_API_URL=https://app.infisical.com \
  sswork/bun-infisical:latest \
  infisical run -- bun your-app.js

Docker Compose

Use a .env file with Docker Compose (recommended):

# docker-compose.yml
version: '3.8'
services:
  app:
    image: sswork/bun-infisical:latest
    env_file:
      - .env
    command: infisical run -- bun start
# .env
INFISICAL_TOKEN=your-service-token
INFISICAL_API_URL=https://app.infisical.com

Kubernetes

Deploy to Kubernetes using secrets:

apiVersion: v1
kind: Secret
metadata:
  name: infisical-secret
type: Opaque
stringData:
  INFISICAL_TOKEN: your-service-token
  INFISICAL_API_URL: https://app.infisical.com
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: bun-app
spec:
  template:
    spec:
      containers:
      - name: app
        image: sswork/bun-infisical:latest
        envFrom:
        - secretRef:
            name: infisical-secret
        command: ["infisical", "run", "--", "bun", "start"]

Self-Hosted Infisical

Connect to your own Infisical instance:

docker run -e INFISICAL_TOKEN=your-service-token \
  -e INFISICAL_API_URL=https://infisical.your-company.com \
  sswork/bun-infisical:latest \
  infisical run -- bun your-app.js

Security Best Practices

🔒

Never Commit Tokens

Never commit your INFISICAL_TOKEN to version control. Always use .env files (added to .gitignore) or secret management systems.

📁

Use .env Files

Store environment variables in .env files during development. Add .env to your .gitignore to prevent accidental commits.

🔄

Rotate Tokens Regularly

Regularly rotate your Infisical service tokens. This limits the impact of potential token leaks and follows security best practices.

🛡️

Least Privilege

Create service tokens with the minimum required permissions. Only grant access to the specific environments and secrets your application needs.

🔍

Audit Access

Regularly review access logs in Infisical. Monitor for unauthorized access attempts and unusual patterns.

📡

Use HTTPS

Always use https:// for INFISICAL_API_URL. Never transmit tokens over unencrypted connections in production.

⚠️ Important Security Notice

If you suspect your INFISICAL_TOKEN has been compromised, revoke it immediately in your Infisical dashboard and generate a new one. Update all deployed applications with the new token.
