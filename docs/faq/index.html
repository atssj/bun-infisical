Frequently Asked Questions

Find answers to common questions about the Bun + Infisical Docker image

General Questions

What is this Docker image?

This is a pre-built Docker image that combines Bun (a fast JavaScript runtime) with the Infisical CLI (secrets management tool). It allows you to run JavaScript/TypeScript applications with secure secrets injection without installing multiple tools separately.

What versions are included in this image?

The current stable release includes:

  • Bun: 1.3.10
  • Infisical CLI: 0.43.58
  • Base Image: Alpine Linux

The image is built on Alpine Linux for minimal size and maximum security.

How do I get an Infisical token?

You can generate an Infisical service token from your Infisical dashboard:

  1. Navigate to your project settings
  2. Go to the Service Tokens section
  3. Create a new token with appropriate permissions for your environment
  4. Copy the token value and set it as the INFISICAL_TOKEN environment variable when running your container

Usage & Deployment

Can I use this image in CI/CD pipelines?

Yes, this image is designed for CI/CD environments. It works seamlessly with:

  • GitHub Actions
  • GitLab CI
  • CircleCI
  • Jenkins
  • Azure DevOps
  • Other popular CI/CD platforms

Use it to run your build and test steps with secrets securely injected from Infisical.

Is this image production ready?

Yes, this image is production-ready. Key features include:

  • Built on Alpine Linux for security and minimal attack surface
  • Follows semantic versioning
  • Includes only stable releases of Bun and Infisical CLI
  • Supports non-root execution
  • Regularly updated with security patches

Technical Details

How is this different from the official Bun image?

The official Bun image (oven/bun) only includes the Bun runtime. This image adds the Infisical CLI, enabling secure secrets management out of the box. You do not need to install or configure Infisical separately, reducing build time and complexity.

How do I update to the latest versions?

Pull the latest tag to get the most recent stable release:

docker pull sswork/bun-infisical:latest

For reproducible builds, use a specific version tag like 1.3.10-0.43.58. Check the Docker Hub page or GitHub releases for available versions and changelog.

What base image is used?

This image is built on Alpine Linux, a security-oriented, lightweight Linux distribution. Alpine provides:

  • Minimal footprint (typically under 10MB base)
  • Reduced attack surface compared to larger distributions
  • Musl libc and busybox for efficiency
  • Package management with apk
