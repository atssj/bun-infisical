Dockerfile Examples

Practical Dockerfile patterns for building applications with Bun and Infisical. From basic setups to optimized multi-stage builds with secure secrets injection.

Basic Dockerfile

A simple Dockerfile for getting started quickly. This example shows the minimal configuration needed to run a Bun application with Infisical.

Dockerfile

# Use the Bun + Infisical base image
FROM sswork/bun-infisical:latest

# Set the working directory
WORKDIR /app

# Copy package files first (for better caching)
COPY package.json bun.lockb ./

# Install dependencies
RUN bun install

# Copy application source code
COPY . .

# Expose the application port
EXPOSE 3000

# Run the application with Infisical secrets injection
CMD ["infisical", "run", "--", "bun", "start"]

Key Points

  • COPY package.json bun.lockb ./ before other files enables Docker layer caching
  • WORKDIR /app creates a dedicated directory for the application
  • EXPOSE 3000 documents the port your app listens on

Multi-Stage Build

Multi-stage builds create smaller, more secure production images by separating build-time dependencies from runtime. This example reduces image size significantly.

Dockerfile (Multi-Stage)

# Stage 1: Build
FROM sswork/bun-infisical:latest AS builder

WORKDIR /app

# Copy and install dependencies
COPY package.json bun.lockb ./
RUN bun install

# Copy source and build
COPY . .
RUN bun run build

# Stage 2: Production
FROM sswork/bun-infisical:latest AS production

WORKDIR /app

# Copy only necessary files from builder
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/package.json ./
COPY --from=builder /app/node_modules ./node_modules

# Create non-root user for security
RUN addgroup -g 1001 -S nodejs && \
    adduser -S bunuser -u 1001

# Change ownership of the app directory
RUN chown -R bunuser:nodejs /app
USER bunuser

# Expose port
EXPOSE 3000

# Run with Infisical
CMD ["infisical", "run", "--", "bun", "dist/index.js"]

Size Comparison

Build Type Typical Size
Basic (single stage) ~150-200 MB
Multi-stage optimized ~80-120 MB

With Infisical Secrets Injection

This example demonstrates proper Infisical configuration for production deployments, including environment variable setup and service token usage.

Dockerfile with Infisical Setup

# Production Dockerfile with Infisical
FROM sswork/bun-infisical:latest

# Set working directory
WORKDIR /app

# Copy dependency files
COPY package.json bun.lockb ./

# Install production dependencies only
RUN bun install --production

# Copy application code
COPY . .

# Create non-root user
RUN addgroup -g 1001 -S nodejs && \
    adduser -S appuser -u 1001 && \
    chown -R appuser:nodejs /app
USER appuser

# Expose application port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD bun -e "console.log('healthy')" || exit 1

# Entrypoint script for Infisical configuration
COPY --chown=appuser:nodejs docker-entrypoint.sh /usr/local/bin/
RUN chmod +x /usr/local/bin/docker-entrypoint.sh

ENTRYPOINT ["docker-entrypoint.sh"]
CMD ["infisical", "run", "--", "bun", "start"]

docker-entrypoint.sh

#!/bin/sh
set -e

# Validate required environment variables
if [ -z "$INFISICAL_TOKEN" ]; then
    echo "Error: INFISICAL_TOKEN is required"
    exit 1
fi

# Set default Infisical API URL if not provided
export INFISICAL_API_URL=${INFISICAL_API_URL:-"https://app.infisical.com"}

echo "Starting application with Infisical..."
echo "API URL: $INFISICAL_API_URL"

# Execute the provided command
exec "$@"

docker-compose.yml

version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    environment:
      - INFISICAL_TOKEN=${INFISICAL_TOKEN}
      - INFISICAL_API_URL=${INFISICAL_API_URL:-https://app.infisical.com}
      - INFISICAL_PROJECT_ID=${INFISICAL_PROJECT_ID}
      - INFISICAL_ENVIRONMENT=${INFISICAL_ENVIRONMENT:-production}
    ports:
      - "3000:3000"
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "bun", "-e", "console.log('healthy')"]
      interval: 30s
      timeout: 3s
      retries: 3

Dockerfile Best Practices

Follow these recommendations to create secure, efficient, and maintainable Docker images.

Layer Caching

Order commands from least to most frequently changing. Copy package files before source code.

# Good - package files cached separately
COPY package.json bun.lockb ./
RUN bun install
COPY . .

Security

Run containers as non-root users. Use the least privileged user necessary.

RUN adduser -S appuser -u 1001
USER appuser

Image Size

Use multi-stage builds and install only production dependencies.

RUN bun install --production

Health Checks

Define HEALTHCHECK instructions for container orchestrators.

HEALTHCHECK --interval=30s \
    CMD bun -e "console.log('ok')"

Secrets Management

Never commit secrets. Use Infisical for runtime injection.

CMD ["infisical", "run", "--", \
     "bun", "start"]

Specific Tags

Pin to specific image versions for reproducible builds.

FROM sswork/bun-infisical:1.3.10-0.43.58
